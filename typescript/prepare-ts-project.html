# 附录一：准备 TypeScript 项目

# 1. 新建项目

  1. 新建项目 ts-syntax，并初始化项目：

    npm init -y

  2. 创建 TypeScript 配置项：

    npx tsc --init

  3. 新建 /src/index.ts 并输入以下内容：

    const lang = 'TypeScript'

console.log(`Hello ${lang}`)

  4. 编译 TypeScript 文件：

    npx tsc src/index.ts

  5. 会在 src 目录下生成编译后的 index.js，内容如下：

    var lang = 'TypeScript'
console.log('Hello ' + lang)

# 2. 配置构建工具

  1. 安装 webpack 及相关工具：

    npm i webpack webpack-cli webpack-dev-server webpack-merge -D

  2. 新建 build 目录，用来存放所有的配置文件，并新建以下四个配置文件：

    build
├── webpack.config.js          # 所有配置文件的入口
├── webpack.base.config.js     # 公共环境的配置
├── webpack.dev.config.js      # 开发环境的配置
└── webpack.pro.config.js      # 生产环境的配置

  3. 编写 webpack.config.js 代码如下：

    const { merge } = require('webpack-merge')
const baseConfig = require('./webpack.base.config')
const devConfig = require('./webpack.dev.config')
const proConfig = require('./webpack.pro.config')

module.exports = (env, argv) =>
  merge(baseConfig, argv.mode === 'development' ? devConfig : proConfig)

    TIP

    为了工程的可维护性，在项目中可以把公共环境、开发环境和生产环境的配置分开来书写，最后通过插件合并。

# 2.1 公共环境配置

  1. 安装 ts-loader：

    npm i ts-loader -D

  2. 安装 html-webpack-plugin：

    npm i html-webpack-plugin -D
    • html-webpack-plugin 可以使用指定的模板生成首页，并且把编译后的 js 文件嵌入到首页。

  3. 新建 src/tpl/index.html 作为首页模板，并输入以下代码：

    <!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>TypeScript Syntax</title>
  </head>

  <body>
    <div id="app"></div>
  </body>
</html>

  4. 修改 webpack.base.config.js 内容如下：

    const HtmlWebpackPlugin = require('html-webpack-plugin')

module.exports = {
  entry: './src/index.ts',
  output: {
    filename: 'app.js',
  },
  resolve: {
    extensions: ['.tsx', '.ts', '.js'],
  },
  module: {
    rules: [
      {
        test: /\.tsx?$/i,
        use: [
          {
            loader: 'ts-loader',
          },
        ],
        exclude: /node_modules/,
      },
    ],
  },
  plugins: [
    new HtmlWebpackPlugin({
      template: './src/tpl/index.html',
    }),
  ],
}

# 2.2 开发环境配置

  1. 修改 webpack.dev.config.js 内容如下：

    module.exports = {
  devtool: 'cheap-module-eval-source-map',
}
    • cheap 忽略文件的列信息；
    • module 定位到 TS 源码，而不是 loader 转义后的 JS 源码；
    • eval-source-map 会把 source map 以 data url 的形式打包到文件中，重编译速度更快。

    有关 source map 的配置说明，可以参阅 https://webpack.js.org/configuration/devtool/ (opens new window)。

  2. 修改 tsconfig.json 打开 Source Map 注释，代码如下：

    "sourceMap": true,

# 2.3 生产环境配置

  1. 安装 clean-webpack-plugin：

    npm i clean-webpack-plugin -D
    • clean-webpack-plugin 会在每次构建之前，清空 dist 目录，从而避免因多次构建产生无用的文件。

  2. 修改 webpack.pro.config.js 内容如下：

    const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
  plugins: [new CleanWebpackPlugin()],
}

# 2.4 修改 npm 脚本

  1. 修改 package.json 的 main 和 scripts 指定入口及构建开发环境的脚本，代码如下：

    "main": "./src/index.ts",
"scripts": {
  "start": "webpack-dev-server --mode=development --config ./build/webpack.config.js",
},
    • main 指定程序的入口文件；
    • start 命令使用 webpack-dev-server 以开发环境启动程序。

  2. 运行 start 命令：

    npm run start
    • 访问 http://localhost:8081/ (opens new window) 并确认控制台能够输出 Hello TypeScript。

  3. 修改 /src/index.ts 内容如下：

    const lang = 'TypeScript'

document.getElementById('app')!.innerHTML = `Hello ${lang}`

  4. 修改 package.json 的 scripts 指定构建生产环境的脚本，代码如下：

    "scripts": {
  "start": "webpack-dev-server --mode=development --config ./build/webpack.config.js",
  "build": "webpack --mode=production --config ./build/webpack.config.js"
},

  5. 构建项目：

    npm run build
