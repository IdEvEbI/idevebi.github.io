# Commit 格式

每次提交，Commit message 都可以包括三个部分：header，body 和 footer，如下所示：

<type>(<scope>): <subject>
// 空一行
<body>
// 空一行
<footer>

其中，header 是必需的，body 和 footer 可以省略。

TIP

为避免自动换行影响美观，commit message 的任何一行都不得超过 72 个字符。

Header 部分只有一行，包括三个字段：type（必需）、scope（可选）和 subject（必需）。

# 1. type

说明 commit 的类别，只允许使用下面 7 个标识。

  • feat: 新功能（feature）
  • fix: 修补 bug
  • docs: 文档（documentation）
  • style: 格式（不影响代码运行的变动）
  • refactor: 重构（即不是新增功能，也不是修改 bug 的代码变动）
  • test: 增加测试
  • chore: 构建过程或辅助工具的变动

TIP

feat 和 fix 的提交应该出现在 Change Log 之中，其他提交则不需要。

# 2. scope

本次提交影响的范围，比如数据层、控制层、视图层等。

# 3. subject

本次提交目的的简短描述，不超过 50 个字符。

  • 以动词开头，使用第一人称现在时，比如change，而不是changed或changes
  • 第一个字母小写
  • 结尾不加句号（.）
