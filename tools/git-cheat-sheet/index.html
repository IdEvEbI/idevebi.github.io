# git 常用命令速查

官方网站：https://git-scm.com/ (opens new window)。

# 一. 配置信息

  1. 设置用户名

    git config --global user.name "用户名"
    • 设置每次提交或打标签时使用的用户名。

  2. 设置邮箱

    git config --global user.email '邮箱地址'
    • 设置每次提交或打标签时使用的邮箱地址。

# 二. 初始化和克隆

  1. 初始化项目

    git init [项目名称]
    • 新建本地仓库，如果指定了 [项目名称]，git 会新建一个名为 项目名称 的目录并在该目录中初始化项目；
    • 如果没有指定 [项目名称]，则会在当前目录初始化项目。

  2. 克隆项目

    git clone [项目 URL]
    • 从远程仓库下载项目及完整的历史记录。

# 三. 日常开发命令

  1. 查看状态

    git status

  2. 对比工作目录和暂存区

    git diff [file]
    • 当想知道对当前对工作目录的文件做了哪些修改时使用。

  3. 对比暂存区和本地仓库

    git diff --staged [file]
    • 当想知道暂存区与本地仓库中的文件有什么区别时使用，通常用在 commit 之前再做一次检查。

  4. 放弃对工作目录文件的修改

    git checkout -- [file]
    • 提示：一旦放弃，无法回退，因为放弃的部分还没有被加入到版本控制。

  5. 添加到暂存区

    git add [file]
    • git add . 会把当前目录（含子目录）中所有修改的文件都添加到暂存区。

  6. 重置文件

    git reset [file]
    • 用暂存区中的文件替换工作目录中的内容。

  7. 提交修改

    git commit -m '提交信息'
    • 创建一次新的提交，把暂存区中的修改提交到本地仓库。

  8. 删除文件

    git rm [file]
    • 从工作目录删除文件，并给暂存区中对应的文件标记为删除。

  9. 将当前修改放入 stash

    git stash
    • 当需要对以前的工作进行修改（例如，发现一个 BUG 急需修复），又不想提交当前工作（例如，当前代码只开发了部分）时使用。

  10. 把 stash 中保存的临时修改恢复到工作目录

    git stash pop
    • 当突发工作完成后，要恢复到之前的工作时使用。

  11. 清空 stash

    git stash drop

# 四. 分支管理

  1. 查看分支列表

    git branch [-a]
    • 默认仅查看本地所有分支，-a 参数会显示所有分支（包含远程）。

  2. 新建分支

    git branch [name]
    • 基于当前工作的分支创建新的分支。

  3. 切换到指定分支

    git checkout [-b] [name]
    • 将当前工作目录切换到指定分支，如果指定的分支不存在，-b 参数会新建一个分支。

  4. 合并分支内容

    git merge [from name]
    • 将 [from name] 分支内容合并到当前分支。

  5. 删除分支

    git brach -d [name]
    • 当已经完成分支合并，不想再继续保留某一分支时使用；
    • 使用 -D 参数可以强制删除分支。

# 五. 检查工作

  1. 查看提交记录

    git log [-n count] --oneline --graph
    • [-n count] 可以查看最近 n 次的提交记录；
    • --oneline 参数会把每次提交显示在一行；
    • --graph 参数会以图形化方式显示日志，查看分支时有用。

  2. 查看操作记录

    git reflog
    • 查看对本地仓库的操作记录，例如：checkout、commit 等。

# 六. 标签管理

  1. 查看所有标签

    git tag

  2. 创建标签

    git tag [name]
    • 通常 [name] 是 v<major>.<minor>.<patch>，其中：
      • major 重大修改
      • minor 小版本修改，minor 相同应该保证彼此兼容
      • patch bug 修复

  3. 为标签添加描述信息

    git tag -a [name] -m "message"
    • -a 参数表示创建的标签能够存储额外的信息（Annonated）；
    • message 可以是发行版本的简要说明，对空开项目有用。

  4. 删除本地标签

    git tag -d [name]

# 七. 撤销修改

  1. 恢复到目标引用

    git reset [--hard] [target ref]
    • 将当前分支切换到 [target ref]，工作目录中与 [target ref] 不一致的内容会被保留并标记；
    • --hard 参数会放弃所有修改。

# 八. 与远程同步

  1. 获取远程修改

    git fetch [remote]
    • 提示：fetch 命令不会把远程修改合并到当前分支。

  2. 获取远程修改并合并到当前分支

    git pull [remote]

  3. 将本地变化推送到远程

    git push [--tags] [remote]
    • --tags 可以推送标签。
